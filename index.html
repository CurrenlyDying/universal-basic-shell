UBS — Universal Basic Shell Command Center

Single-page control panel for setup, configuration, routing tests, generator, and operations.

UBS architecture

  • Tier 0: Apps Script proxy running from Google infra.
  • Tier 1: Cloud Run signaling server + Colab/Cloud Shell browser worker node.
  • Tier 2: Headscale/WireGuard device tunnel.
  • Tier 3: Volunteer residential nodes.

This page is designed to be your center of command: configure once, then run all checks and workflows.

Step-by-step setup (in-page)

  1. Create Apps Script URL: paste `apps-script/proxy.gs` + `apps-script/gemini.gs` into script.google.com, set `GEMINI_KEY`, deploy Web App as Execute as Me and Anyone, then copy URL into Command Center.
  2. Deploy Cloud Run signaling: in Cloud Shell, clone repo and cd to repo root before running gcloud run deploy --source ./container ... (fixes could not find source [./container]), verify /health, then paste URL into Command Center.
  3. Deploy Cloudflare Worker: create KV namespace, copy `worker/wrangler.toml.example` to `worker/wrangler.toml`, fill values from Command Center snippet, deploy with Wrangler.
  4. Start Colab worker node: open `colab/default.ipynb`, set `UBS_SIGNAL_URL` from Command Center, run all cells, verify register + heartbeat logs.
  5. Run live checks: use the Live Checks tab to verify signal health, active nodes, and Tier 1 assignment.
